# 《异界旅人》原生堆叠卡牌交互修正说明 v0.4

## 1. 修正原因

v0.3 把概念图中的地点卡、资源卡和角色卡实现成了全屏 Canvas 中的 `Button`。这虽然能完成点选式验证闭环，却覆盖在项目原有的 3D 卡牌桌面之上，并通过 `GraphicRaycaster` 抢走指针事件，导致真正的 `CardController` 无法收到拖拽输入。

这与项目原本的核心玩法方向不一致。项目已有的正确交互链是：

```text
拖起一张或一叠原生卡牌
→ 在桌面移动
→ 放到兼容卡牌附近
→ CardStack 合并
→ CraftingManager 检查配方或触发探索、采集、战斗
```

## 2. 本次修正

- 中央区域重新交给 StackCraft 原生 3D 卡牌桌面。
- `WorldMapView` 与 `ForestView` 只保留为透明、不可射线的状态容器。
- 删除中央区域所有按钮式地点卡、资源卡和角色卡。
- HUD 只保留在顶部、左下、右侧和底部边缘，不再遮挡桌面中心。
- 底部明确提示“拖动卡牌并叠放以互动”。
- 配方卡只用于查看，不再通过点击直接制作。
- `SaveButton` 与 `LoadButton` 保留内部引用，但默认隐藏，不在当前正式界面露出。

## 3. 当前正确操作方式

原生卡牌继续使用项目已有的操作：

- 拖动角色卡到资源卡上进行采集或加工。
- 拖动角色卡到森林等区域卡上开始探索。
- 将材料、角色、建筑等卡牌按配方叠放，由卡堆触发制作。
- 拖动卡堆中的中间卡会从该位置拆分卡堆。
- 将兼容卡堆靠近并放下会自动合并。
- 将角色叠到敌人附近继续使用现有战斗流程。

## 4. 保留与后续工作

连续世界时间、卡牌容器数据和统一存档代码仍然保留，但不再用按钮式假卡驱动玩法。下一步应把连续时间接到原生 `CraftingTask`、探索配方与卡堆事件上，使“卡牌叠放开始行动，行动推进世界时间”成为同一套系统。

两张生成的地图图像仍保留为美术资产，但当前不再作为 Screen Space Overlay 覆盖原生卡牌。后续如要使用，应贴到 3D `Board` 材质或世界空间背景上，确保它位于卡牌下方。
