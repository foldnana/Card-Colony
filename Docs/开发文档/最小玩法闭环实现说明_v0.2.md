# 《异界旅人》最小玩法闭环实现说明 v0.2

## 1. 本次目标

本次把上一阶段的纯领域模型接入 Unity，形成可以在 `Main` 场景中亲自操作的最小闭环：

```text
探索低语森林 → 采集草药 → 草药卡进入背包 → 消耗草药制作药水 → 药水卡进入背包
```

同时接入行动驱动的连续时间、玩家背包卡牌界面，以及覆盖世界时间、世界状态、背包和进行中行动的统一运行快照。

现有 StackCraft 自动战斗代码没有改动。新界面启动时会停用旧 `TimeManager` 的空闲计时，并保持 `Time.timeScale = 1`，因此原实时自动战斗仍按正常实时速度运行。

## 2. 玩家操作

打开 `Assets/StackCraft/Scenes/Main.unity` 并进入 Play Mode 后，会显示新的 Card Colony 界面。

### 时间控制

- `暂停/继续`：只暂停或继续新的世界时钟。
- `正常`：世界时间按正常速度推进。
- `四倍速`：世界时间和当前行动按四倍速度推进。
- `主动等待/停止等待`：没有行动时，玩家可以主动让连续时间流逝。
- 没有行动且没有主动等待时，世界时间保持不变。

界面以 `第N天 HH:mm` 显示连续时间，不再使用上午、下午和夜晚三个离散阶段。

### 最小玩法链

1. 点击`探索低语森林（30 分钟）`。
2. 探索结束后，点击`采集草药（20 分钟）`。
3. 背包获得三份野生草药，并显示为一张数量为 3 的物品卡。
4. 点击`制作治疗药水（2 草药 / 15 分钟）`。
5. 开始制作时扣除两份草药；制作结束后获得一张治疗药水卡。

所有行动使用世界分钟作为进度，因此四倍速会缩短现实等待时间，但消耗的世界时间保持一致。

### 背包操作

背包直接遍历 `CardContainer.Cards` 生成 `ItemCardView`：

- 卡面显示物品名称、数量、品质、总重量和批次。
- `取出整组`把整张物品卡放到手持区。
- `拆分 1 张`从数量大于 1 的卡牌中拆出一个物品到手持区。
- `放回背包`把手持卡重新放入背包，并按容器规则合并。
- 背包顶部显示槽位和承重占用。
- 手持卡属于统一运行状态，退出或读取后不会凭空丢失。
- 行动进行中不能取出或拆分卡牌；行动开始前会检查产物所需空间。

## 3. 连续时间与行动

`PlayableLoopSession` 统一持有：

- `ActionDrivenWorldClock`
- `CardContainer PlayerInventory`
- `WorldStateStore`
- 当前 `LoopActionState`

当前实现了三种行动：

| 行动 | 世界耗时 | 完成效果 |
|---|---:|---|
| 探索低语森林 | 30 分钟 | 发现地点并写入探索进度与世界标记 |
| 采集草药 | 20 分钟 | 背包获得 3 份野生草药 |
| 制作治疗药水 | 15 分钟 | 开始时消耗 2 份草药，完成时获得 1 瓶药水 |

行动开始时会向世界时钟注册行动句柄，完成后释放。空闲时没有句柄，因此不会错误推进时间。

## 4. 统一存档

`RunSnapshot` 统一保存：

- 累计世界分钟。
- 暂停、速度和主动等待状态。
- 世界标记、数值与地点状态。
- 玩家背包容量及所有物品卡字段。
- 当前手持卡及其数量、品质、重量、批次和实例 ID。
- 当前行动类型、稳定 ID、总耗时和已完成进度。

界面的`保存进度`和`读取进度`使用 `RunSnapshotFileStore`，保存到：

```text
Application.persistentDataPath/CardColonyPlayableLoop.json
```

写入时先把临时文件完整刷新到磁盘，再原子替换正式存档，同时保留上一份 `.bak` 备份。主存档损坏时会尝试读取备份；读写失败会在界面显示原因，并保持当前会话不变。读取进行中制作时不会再次扣除草药，完成后也只会发放一次产物。

## 5. UI 资产结构

UI 使用场景/Prefab 骨架与运行时动态卡牌的混合方式：

```text
GameUiRoot.prefab
├─ TopBar
│  ├─ TimeText
│  ├─ StatusText
│  └─ 时间控制按钮
├─ ActionPanel
│  ├─ ActionProgressText
│  ├─ 探索/采集/制作按钮
│  └─ 保存/读取按钮
└─ BackpackPanel
   ├─ InventorySummaryText
   ├─ HeldCardText
   └─ InventoryScroll/Viewport/Content
      └─ ItemCardView.prefab（运行时生成）
```

关键资产：

- `Assets/CardColony/Prefabs/GameUiRoot.prefab`
- `Assets/CardColony/Prefabs/ItemCardView.prefab`
- `Assets/CardColony/Scripts/Editor/CardColonyUiPrefabBuilder.cs`

需要重新生成 Prefab 时，可使用 Unity 菜单：

```text
Tools > Card Colony > 重建概念图风格可玩界面
```

## 6. 主要代码

### 领域层

- `Gameplay/PlayableLoopSession.cs`：完整玩法链、行动推进和统一快照。
- `Gameplay/LoopActionState.cs`：可保存的进行中行动。
- `Gameplay/RunSnapshot.cs`：统一运行快照结构。
- `Inventory/CardContainerSnapshot.cs`：容器与物品卡快照。
- `Presentation/BackpackInteraction.cs`：取出、拆分和放回。
- `Presentation/WorldClockTextFormatter.cs`：日期与 24 小时时刻格式化。

### Unity 接入层

- `WorldClockDriver.cs`：每帧驱动、暂停/速度/等待、保存与加载。
- `UI/GameUiPresenter.cs`：按钮绑定、状态显示和动态背包刷新。
- `UI/ItemCardView.cs`：单张物品卡显示与操作。
- `Save/RunSnapshotJsonSerializer.cs`：JSON 序列化。
- `Save/RunSnapshotFileStore.cs`：实际文件保存、读取与清理。

## 7. 测试范围

测试覆盖：

- 探索、采集、制作整条领域链。
- 空闲、等待、暂停、正常速度和四倍速。
- 进行中行动保存与恢复，避免重复扣除和重复产出。
- 背包跨卡消耗、事务式奖励、容量预检、快照恢复、整组取出、拆分与放回。
- 手持卡保存恢复，以及行动期间禁止改变背包容量。
- JSON 与实际文件存档往返、覆盖备份、损坏文件失败感知与备份恢复。
- `GameUiRoot.prefab` 的真实按钮点击闭环和动态卡牌数量。
- `Main` 场景 Prefab 接入。
- 停用旧空闲计时器且保持自动战斗所需的正常实时速度。

## 8. 当前边界

- 这是用于验证核心体验的功能型 UI，尚未加入正式图标、动效和音效。
- 当前只有低语森林、草药和治疗药水这一组演示内容。
- 当前在行动开始前预留产物空间；背包不足时不会开始行动，也不会扣除制作材料。加入仓库后可进一步演进为独立待领取奖励区。
- 统一快照目前独立于旧 StackCraft 多槽位存档。后续可把 `RunSnapshot` 挂入正式角色存档槽，而不再使用固定演示文件名。
- 马车、仓库和跨容器事务式转移尚未接入，但可以复用 `CardContainer` 和 `ItemCardView`。
