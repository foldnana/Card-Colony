# 新游戏世界地图接入说明 v0.5

## 本次目标

把“新游戏仍然进入旧 Starter Pack 玩法”的启动问题修正为：从标题界面创建新游戏后，直接进入概念图风格的世界地图，并保留 StackCraft 原生卡牌的拖动、叠放、拆分与配方交互。

## 启动流程调整

- `Main` 场景不再把 `00_Pack_Starter` 配置为新游戏默认生成物。
- 新增 `WorldMapBootstrap`，仅在创建新场景数据时生成世界地图卡组。
- 读取已有存档时不重复生成地图卡，而是继续由 `CardManager` 恢复保存的卡堆。
- `GameUiPresenter` 现在允许先于 `WorldClockDriver` 完成会话初始化，修复首次进入场景时可能出现的空引用。

## 世界地图卡组

新游戏会按固定位置创建 10 张原生卡牌：

- 6 张地点卡：河湾村、白石城、旧矿洞、低语森林、马车、魔王城。
- 1 张旅行者卡。
- 3 张职业卡：战士、游侠、法师；出生时组成一个可拖动、可拆分的原生卡堆。

这些卡牌全部通过 `CardManager.CreateCardInstance` 创建，仍然使用 `CardController` 和 `CardStack`，不是 UI 按钮或静态图片。

其中低语森林继续使用原有 `Forest` 定义，旅行者继续使用原有 `Villager` 定义。因此把旅行者拖到低语森林上，仍可匹配现有探索配方并推进原生探索流程。

## 地图与 HUD

- 世界地图插画应用到原场景背景平面，不创建会拦截射线的新碰撞体。
- 关闭旧棋盘的灰色外观，但保留 `Board` 组件负责边界计算和卡牌放置规则。
- HUD 改为概念图的边缘式布局：
  - 左上：日期、具体时刻和播放/暂停。
  - 顶部：角色状态以及正常、四倍速、主动等待。
  - 左下：旅行者状态和卡牌背包。
  - 右侧：任务/配方页签。
  - 底部：互动、扎营、地图导航。
- 保存、加载按钮仍保留内部绑定，但默认隐藏，不在当前游戏界面露出。
- 屏幕中央不放置可射线命中的 UI，原生 3D 卡牌可以正常收到拖拽事件。

## 验证结果

- 从 `Title` 场景调用新游戏，成功进入 `Main`。
- 运行时检测到 10 张原生卡牌以及有效的 `WorldMapBootstrap`。
- 世界地图、6 张地点卡、旅行者和职业卡堆均在相机可视范围内。
- 启动日志中不再出现 `GameUiPresenter`/`WorldClockDriver` 初始化空引用。
- 新增回归测试覆盖：
  - `Main` 不再引用 Starter Pack。
  - 世界地图启动器的地点、旅行者、职业卡堆和地图资源配置完整。
  - HUD 使用概念图边缘布局。
  - UI 中央区域不拦截原生卡牌射线。
  - UI 可在世界时钟会话尚未初始化时安全绑定。

## 当前可玩入口

1. 从标题界面选择新游戏。
2. 进入世界地图后，直接拖动旅行者、地点或职业卡堆。
3. 把旅行者叠放到低语森林，触发当前已有的探索配方。
4. 使用顶部按钮控制暂停、正常速度、四倍速和主动等待。

后续建议优先把河湾村、旧矿洞、白石城、马车和魔王城分别接入各自的旅行或交互配方；本次先完成“进入新系统、看见世界地图、原生卡牌可操作”的正确入口。

## 当前边界

- 世界地图中央使用 StackCraft 原生卡牌与卡堆，已经支持原生拖拽、叠放和拆分。
- 左下角背包仍绑定早期 `CardContainer` 演示数据；原生探索掉落还没有自动汇入这套背包。因此当前版本不把两套容器宣称为已经统一。
- 下一步应建立 StackCraft `CardInstance/CardStack` 与玩家权威容器之间的单一数据流，让探索掉落、制作消耗、卡牌背包和存档都读取同一份物品状态。
