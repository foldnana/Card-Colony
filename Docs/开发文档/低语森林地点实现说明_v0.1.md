# 低语森林地点实现说明 v0.1

## 地点入口

- 世界地图地点 ID：`whispering-forest`
- 复用场景：`Assets/StackCraft/Scenes/Location.unity`
- 地点定义：`Assets/StackCraft/Resources/Locations/Location_WhisperingForest.asset`
- 安装菜单：`Tools > Card Colony > Install Whispering Forest`

世界地图的小队停靠在低语森林后，可以通过右侧地点面板进入。返回时仍回到世界地图，并保留队伍成员的生命、饱食度和装备数据。

## 地图美术

正式背景使用 `WhisperingForestBackground_v2.png`。画面保持河湾村的灰蓝纸张、低饱和简笔线稿风格，并在林间道路和空地周围保留卡牌摆放空间。旧版 `WhisperingForestBackground.png` 继续保留，不被覆盖。

## 随机内容

低语森林使用通用地点随机生成规则。每次从世界地图进入时，系统会重新生成下列原项目卡牌，并改变数量与布局：

| 卡牌 | 每次生成数量 |
| --- | ---: |
| 石块 | 2–4 |
| 浆果丛 | 2–4 |
| 树木 | 3–6 |
| 史莱姆 | 1–3 |
| 哥布林 | 0–2 |

布局使用独立随机种子，在 13.2 × 5.4 的地图区域中布置，并尽量保持 1.05 个世界单位的间距。区域已避开顶部 UI 禁放带，玩家所有队员周围均预留 2.1 个世界单位的安全区。

## 存档与重新进入

随机生成卡牌带有 `LocationRandomSpawnMarker`，该标记会写入 `CardData`：

- 在地点中直接读档：保留存档中的随机卡牌，不刷新资源和敌人。
- 返回世界地图后再次进入：删除上次带标记的随机卡牌，再生成新一批。
- 进入建筑、洞穴或下一层后再返回父地点：不刷新父地点，避免利用子地点往返反复重抽资源。
- 玩家自行带入或留在地图上的普通卡牌不会被刷新逻辑删除。
- 随机敌人掉落的战利品没有随机地点标记，因此不会被当作下一轮环境内容清理。
- 如果离开地点时仍在战斗，重进前会先通过战斗系统正式结束旧战斗，再清理旧随机敌人和替换队伍，避免战斗卡脱离普通堆栈后残留。

## 复用到后续野外地点

后续矿洞、遗迹等地点可在 `LocationTemplate` 中配置：

- `RandomizeCardsOnEntry`
- `RandomSpawnAreaCenter`
- `RandomSpawnAreaSize`
- `RandomSpawnMinSpacing`
- `RandomSpawnPartyClearance`
- `RandomCardSpawns`

数量和布局生成逻辑统一由 `LocationSceneController` 处理，具体地点安装器只提供数据，不需要复制一套随机生成代码。
