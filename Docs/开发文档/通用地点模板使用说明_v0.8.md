# 通用地点模板使用说明 v0.8

## 目标

河湾村、河湾旅馆以及后续的低语森林、矿洞、遗迹和城镇统一复用
`Assets/StackCraft/Scenes/Location.unity`。新地点不再复制 Unity 场景，也不需要把
地点定义手工写进场景数组。

## 运行时结构

- `LocationDefinition` 保存地点名称、背景、地图尺寸、镜头参数、玩家出生点、初始
  卡牌和建筑入口。
- `LocationSceneController` 从场景中已有定义和
  `Resources/Locations` 自动发现的定义构建地点目录。
- 相同地点 ID 只保留一个定义；场景中显式配置的旧定义优先，保证已有存档和场景
  引用稳定。
- 地图背景、边界、镜头缩放、初始卡牌、玩家队伍和返回逻辑均由通用场景处理。

## 创建地点

在 Unity 菜单选择：

```text
Tools > Card Colony > Create Reusable Location
```

工具会在以下目录创建一份新的地点定义并自动选中：

```text
Assets/StackCraft/Resources/Locations/
```

在 Inspector 中填写：

1. `Id`：稳定且唯一的英文 ID，例如 `whispering-forest`。
2. `Display Name`：中文地点名。
3. `Background Texture`：地点背景图。
4. `Map Size`：卡牌活动区域和背景共同使用的世界尺寸。
5. 镜头最小、最大、初始距离和滚轮缩放速度。
6. 玩家出生点和队员间距。
7. `Initial Card Spawns`：卡牌定义和固定出生坐标。
8. `Entrances`：只有地点内部还能进入建筑或下一层时才填写。
9. 野外地点可开启 `Randomize Cards On Entry`，并填写随机区域、最小间距、出生点
   安全距离和 `Random Card Spawns` 的卡牌数量范围。

地点资产只要位于 `Resources/Locations` 下，就会被通用地点场景自动发现。

## 代码生成地点

需要由安装器批量创建内容时，使用 `LocationTemplateBuilder.CreateOrUpdate`。河湾村
和河湾旅馆已经迁移到该入口，后续地点安装器只负责提供数据：

```csharp
LocationTemplateBuilder.CreateOrUpdate(
    definitionPath,
    new LocationTemplate
    {
        Id = "whispering-forest",
        DisplayName = "低语森林",
        BackgroundTexture = background,
        MapSize = new Vector2(22f, 14f),
        InitialCardSpawns = spawns,
        RandomizeCardsOnEntry = true,
        RandomSpawnAreaSize = new Vector2(18f, 9f),
        RandomCardSpawns = randomSpawns
    });
```

更新地点时会按建筑卡追加或更新模板声明的入口，同时保留其他系统后来添加的入口；
通用模板不会整体清空入口列表。

## 当前边界

模板负责地点承载、固定内容和按数量范围生成随机环境卡牌，不负责具体玩法。药草
采集、资源枯竭、任务进度和战斗仍应做成可以跨地点复用的独立系统，不能写死在
低语森林安装器中。
